# SlideForge Mac

SlideForge Mac is a native macOS tool for converting SDPC whole-slide files into OpenSlide-readable WSI files.

SlideForge Mac 是一款 macOS 原生工具，用于将 SDPC 全切片病理图像转换为 OpenSlide 可读取的 WSI 文件。

Created by Xinyu Ge.

由 Xinyu Ge 制作。

It reads SDPC tile tables directly, decodes HEVC/H.265 tile payloads with PyAV/FFmpeg, and writes tiled pyramidal BigTIFF with an optional `.svs` hardlink or symlink entry point.

本工具直接读取 SDPC 瓦片表，使用 PyAV/FFmpeg 解码 HEVC/H.265 瓦片数据，并写出带金字塔层级的 tiled BigTIFF；也可以额外生成 `.svs` 硬链接或软链接入口，便于 WSI 工作流读取。

## Features / 功能

- Native macOS SDPC parsing, no Windows DLL and no Linux `.so`
- macOS 原生解析 SDPC，不依赖 Windows DLL 或 Linux `.so`
- Converts `.sdpc` to OpenSlide-readable `.ome.tif`
- 将 `.sdpc` 转换为 OpenSlide 可读取的 `.ome.tif`
- Creates `.svs` entry points for WSI workflows
- 可为 WSI 工作流创建 `.svs` 入口
- Safe preview generation without expanding the full slide into memory
- 支持安全预览，不会把整张大图一次性展开到内存
- GUI and CLI entry points
- 同时提供图形界面和命令行入口

## Install / 安装

```bash
brew install ffmpeg openslide
python3 -m pip install -e .
```

If you only need the command line:

如果只需要命令行：

```bash
python3 -m pip install -r requirements.txt
```

## Command Line / 命令行

```bash
slideforge-convert input.sdpc output.ome.tif --compression jpeg --jpeg-quality 90
```

For filenames that contain spaces, Chinese characters, or other non-ASCII text, wrap both paths in quotes.

如果文件名包含空格、中文或其他非 ASCII 字符，请用英文双引号包住输入和输出路径。

```bash
slideforge-convert "6213966 癌.sdpc" "6213966.ome.tif"
```

Recommended output names:

推荐输出命名：

- Use `.ome.tif` for the converted pyramidal WSI file.
- 转换后的金字塔 WSI 文件建议使用 `.ome.tif`。
- Use `.svs` only as an additional hardlink or symlink entry point after conversion.
- `.svs` 建议只作为转换后的额外硬链接或软链接入口。

Create an `.svs` entry point after conversion:

转换完成后创建 `.svs` 入口：

```bash
ln "6213966.ome.tif" "6213966.svs"
```

If you see an error such as `unrecognized arguments`, the input filename was probably split by the shell. Add quotes around filenames with spaces.

如果看到 `unrecognized arguments` 这类错误，通常是文件名被终端按空格拆开了。请给带空格的文件名加英文双引号。

```bash
slideforge-convert "cancer.sdpc" "cancer.ome.tif"

# Or / 或者
slideforge-convert "cancer.sdpc" "cancer.svs"
```

## GUI / 图形界面

```bash
slideforge-gui
```

You can also launch the visual interface from the repository:

也可以从仓库目录直接启动图形界面：

```bash
chmod +x scripts/launch_gui.command
./scripts/launch_gui.command
```

To create a double-clickable macOS app bundle:

生成可双击打开的 macOS App：

```bash
python3 scripts/create_macos_app.py
open "dist/SlideForge Mac.app"
```

The GUI supports:

图形界面支持：

- selecting SDPC or WSI files
- 选择 SDPC 或 WSI 文件
- safe preview
- 生成安全预览
- converting SDPC to WSI
- 将 SDPC 转换为 WSI
- stopping a long conversion
- 停止耗时较长的转换任务

## Important Usage Note / 重要使用说明

Do not open huge `.ome.tif` or `.svs` files with macOS Preview. Use OpenSlide, QuPath, your model pipeline, or the safe preview JPG generated by SlideForge.

不要用 macOS 自带“预览”直接打开巨大的 `.ome.tif` 或 `.svs` 文件。请使用 OpenSlide、QuPath、你的模型流程，或使用 SlideForge 生成的安全预览 JPG。

## Current Format Support / 当前格式支持

This project currently targets the observed SDPC variant with:

本项目目前支持已观察到的 SDPC 变体，其特征包括：

- `SQ` file header
- `SQ` 文件头
- `IFz` tile-layer records
- `IFz` 瓦片层级记录
- HEVC/H.265 tile payloads
- HEVC/H.265 瓦片数据
- 672-pixel tiles in the tested files
- 测试文件中的瓦片尺寸为 672 像素

Other SDPC variants may need additional parsing support.

其他 SDPC 变体可能需要额外适配。
